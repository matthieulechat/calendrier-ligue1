# ZBLK-20260702232209-1 — Impression sur 2 pages : l'affiche dépassait 297mm (footer relégué sur une 2e page quasi vide)

| Friction                                                                                                                                                                                                  | Cause réelle                                                                                                                                                                                                                                                                                                                                                                                              | Solution                                                                                                                                                                                                                                                                                   | Statut |
| --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| L'aperçu d'impression Chrome montrait 2 feuilles de papier : la 2e ne contenait que le footer (texte légal + logo), le reste blanc — repéré via des captures d'écran de la boîte de dialogue d'impression | `min-h-[297mm]` sur le conteneur `PosterSheet` laisse le contenu déborder si sa hauteur réelle dépasse 297mm — mesuré empiriquement via `agent-browser` (`getBoundingClientRect()` converti en mm) à **307,12mm**, soit 10,12mm de trop, confirmant précisément la marge d'erreur déjà notée dans `docs/ROADMAP.md` §1.3 pour la colonne la plus dense (18 lignes/5 mois avec les données fake actuelles) | Resserré `MatchRow` (`py-[0.9mm]` → `py-[0.6mm]`, gain ~0,6mm/ligne × jusqu'à 18 lignes) et `MonthBlock` (`mt-[2.5mm]` → `mt-[2mm]` entre blocs de mois) — remesuré après coup : contenu réel à 286,4mm, soit 10,6mm de marge sous 297mm, couvrant aussi le cas réel V1 (18 matchs/6 mois) | résolu |
