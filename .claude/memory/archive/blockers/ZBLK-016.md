# ZBLK-016 — `git reset --hard` a effacé une consolidation mémoire non commitée

| Friction                                                                                                                                                                                                                                                                                                                                                                                                             | Cause réelle                                                                                                                                                                                                                                                                                                                    | Solution                                                                                                                                                                                                                                                                                                                                                                                                                                      | Statut |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| Un `git reset --hard` fait par Baptiste pour nettoyer des commits de test non désirés a aussi effacé, comme effet de bord, des modifications de fichiers mémoire (`blockers.md`, `decisions.md`, `learnings.md`, `journal.md`, plusieurs fichiers individuels `BDR-004`/`BDR-007`/`BDR-011`/`LRN-016`/`LRN-017`/`LRN-018`) qui étaient dans le répertoire de travail depuis une session antérieure, jamais commitées | `git reset --hard` s'applique à **tous** les fichiers trackés modifiés dans le répertoire de travail, pas seulement à ceux liés à l'action en cours — une consolidation mémoire sans rapport avec le travail de la session (Phase 6) partageait le même répertoire de travail non commité et a été discardée sans avertissement | Repéré immédiatement après le reset (`git diff HEAD` vide sur les fichiers concernés, aucun stash), signalé à Baptiste sur le moment. Les nouveaux fichiers non trackés de cette ancienne consolidation ont survécu (untracked ≠ affecté par `reset --hard`) et ont permis une récupération quasi complète en fin de session via ce rituel de fermeture (BDR-004 recorrigée, BDR-012/LRN-019-021/BLK-013 réindexés, BLK-010/011/012 archivés) | résolu |

## Références

- Récupération effectuée dans ce même rituel `/memory-close` du 2026-07-18
