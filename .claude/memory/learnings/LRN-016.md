# LRN-016 — Trancher un conflit de merge "côté entier" risque de réintroduire un choix abandonné

| Pattern observé                                                                                                                                                                                                                                                                                                                                                                                                              | Contexte                                                                                                                                                                                                                                                                                                   | Application future                                                                                                                                                                                                                                                                        |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Résoudre un conflit de merge en gardant un côté "en bloc" (HEAD ou incoming entier) traite le hunk comme une alternative binaire — si l'un des deux côtés mélange en réalité plusieurs changements indépendants (ex. une amélioration technique + un choix visuel), le choix en bloc réintroduit silencieusement celui des deux qui avait été abandonné par l'autre branche, sans qu'aucun diff ne le signale comme anormal. | Rebase de `phase-1` sur `origin/main` : conflit sur `ScoreBox.tsx` entre une version interactive (Phase 5) sans libellé visible et une version statique avec libellé — le choix "garder HEAD" a réintroduit le libellé alors qu'il avait été retiré délibérément ailleurs dans la même session historique. | Avant de résoudre un conflit en gardant un côté en bloc, vérifier si le journal/mémoire du projet documente une décision plus récente sur UN SEUL des changements mélangés dans ce hunk — dans ce cas, reconstruire manuellement en combinant les deux plutôt que choisir un camp entier. |
