# LRN-020 — football-data.org : date/heure placeholder identique pour toute une journée tant que non resynchronisée

| Pattern observé                                                                                                                                                                                                                                                                                                                        | Contexte                                                                                                                                                                                                           | Application future                                                                                                                                                                                                                                                                                                                                               |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Les 9 matchs de la journée 1 (2026-08-22, dans ~5 semaines au moment du test) partagent tous exactement le même `utcDate` (`2026-08-22T15:00:00Z`) côté football-data.org, alors que ligue1.com affiche déjà les vraies dates/heures par match — le fournisseur n'a pas encore resynchronisé le calendrier officiel pour cette journée | Vérifié en interrogeant directement `/competitions/FL1/matches?matchday=1&season=2026` (indépendamment d'un éventuel bug de `sync-matches.ts`) — la valeur vient bien de l'API, pas d'une erreur de transformation | Ne pas considérer les dates football-data.org comme définitives pour une journée lointaine — re-lancer `pnpm sync-matches` plus près de chaque journée (les horaires/droits TV se confirment progressivement). Le poster n'affiche que la date/jour de semaine (pas d'heure, cf. `MatchRow.tsx`), donc seul le jour compte pour ce projet, pas le kickoff précis |

- Décision (2026-07-18) : accepter la limitation, pas de scraping complémentaire de ligue1.com pour l'instant — re-sync périodique suffit
