# LRN-021 — Guider un utilisateur à ajouter une clé API : vérifier que `.env.example` reste un placeholder

| Pattern observé                                                                                                                                                                                                                                                                                                        | Contexte                                                                                                                                                                                              | Application future                                                                                                                                                                                                                                                                  |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Après avoir suivi un guide pointant explicitement vers `.env.local` pour stocker une clé API, la vraie clé s'est retrouvée dans `.env.example` (non gitignoré, contrairement à `.env`/`.env.local`) — deux valeurs différentes sont même apparues entre `.env` et `.env.example`, signe de confusion ou d'autofill IDE | Mise en place de `FOOTBALL_DATA_ORG_KEY` pour `scripts/sync-matches.ts` (Phase 6, calendrier-ligue1) — repéré via `git status` (fichier untracked, donc pas encore de fuite réelle) avant tout commit | Après toute étape qui demande à l'utilisateur d'ajouter un secret dans un fichier `.env*`, vérifier explicitement le contenu de `.env.example` avant un commit/push — ne jamais supposer que le guide a été suivi à la lettre, quel que soit l'outil (IDE, copier-coller, autofill) |
