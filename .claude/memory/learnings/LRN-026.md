# LRN-026 — Logo club sur fond de sa propre couleur = risque de contraste ; le pattern "pastille blanche" existait déjà, juste pas partout

| Pattern observé                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | Contexte                                                                                                                                                                                                                                                                                                                                          | Application future                                                                                                                                                                                                                                                                                                                                                                                       |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Les couleurs de fond (`--club-secondary`) sont extraites du logo lui-même via `colorthief` (cf. [BDR-006](../decisions/BDR-006.md)). Le script garantit le contraste **texte blanc / fond**, mais rien ne garantit le contraste **logo / fond** — si le logo est monochrome ou dominé par une seule teinte (cas du nouveau blason OM 2026, très majoritairement bleu), le fond extrait de ce même logo peut être une teinte de bleu très proche, rendant le logo quasi invisible une fois posé dessus. | Après le remplacement du logo OM ([LRN-025](LRN-025.md)) et régénération des couleurs, seule l'affiche OM devenait illisible : logo bleu sur bandeau bleu (`--club-secondary` #005894 vs logo dominante #005da4) — spécifique à ce club, les autres logos ont assez de blanc/contraste interne pour rester lisibles sur leur propre fond extrait. | Après tout remplacement de logo suivi de régénération de couleurs (`scripts/extract-club-colors.ts`), vérifier visuellement (agent-browser) le rendu du header pour le club concerné — un logo mono-teinte peut faire ressortir ce problème même si le script de contraste texte passe sans erreur.                                                                                                      |
| `MatchRow.tsx` appliquait déjà une pastille blanche (`rounded-full bg-white/[92%] p-[0.4mm]`) derrière les logos adverses dans les lignes de match — mais `PosterHeader.tsx` affichait le logo du club sélectionné brut, sans backing, directement sur `bg-[var(--club-secondary)]`. Incohérence entre deux composants du même poster, pas un manque de pattern à inventer.                                                                                                                            | Fix initial (trop large) : reprise du même traitement (`rounded-full bg-white/[92%] p-[1.5mm] object-contain` + `print-color-adjust:exact`) appliqué inconditionnellement sur le logo du header pour tous les clubs.                                                                                                                              | Avant d'inventer un traitement de lisibilité (masque, ombre, contour), grep les usages existants de `club.logo`/`opponent.logo` dans le repo — un pattern de contraste peut déjà exister ailleurs dans le même poster et n'avoir simplement pas été appliqué partout.                                                                                                                                    |
| Baptiste a recadré : la pastille blanche ne doit s'appliquer qu'à OM (seul club concerné), pas à tous — un traitement universel change le rendu de clubs qui n'en ont pas besoin. Passage à un flag data-driven `Club.logoNeedsBackdrop?: boolean` dans `clubs.json`/`types/match.ts`, lu conditionnellement dans `PosterHeader.tsx` **et** `PosterFooter.tsx` (même logo club affiché une 2e fois en bas à droite de l'affiche, même bug potentiel, repéré via `grep club.logo`).                     | `"logoNeedsBackdrop": true` ajouté uniquement à l'entrée `marseille` de `clubs.json`. Les deux composants appliquent la classe pastille seulement si `club.logoNeedsBackdrop`. Vérifié visuellement : OM a la pastille sur header+footer, Le Mans FC reste sans backing.                                                                          | Une lisibilité spécifique à un club = flag manuel sur ce club dans les données (`logoNeedsBackdrop`), pas un calcul de contraste logo/fond généralisé (over-engineering pour un seul cas actuel) ni un traitement universel (changerait le rendu de tous les autres clubs sans besoin). Si un 2e club a le même problème un jour, ajouter le flag sur son entrée — pas de nouvelle abstraction avant ça. |
