# LRN-018 — `TaskStop` ne garantit pas la mort d'un process Bash arrière-plan ; `robocopy /MIR` pour purger un `node_modules` pnpm à chemins longs

| Pattern observé                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              | Contexte                                                                                                                         | Application future                                                                                                                                                                                                                                                                                                                                                     |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `TaskStop` sur un job Bash lancé en arrière-plan via `(commande &)` dans un sous-shell avec redirection de sortie peut répondre "Successfully stopped" sans que le process réel (`cp`, ici) soit effectivement terminé — il continue d'écrire des fichiers en tâche de fond. Par ailleurs, l'arborescence `.pnpm/<pkg>@version/node_modules/...` de pnpm dépasse souvent la limite de chemin Windows (260 caractères), faisant échouer `rm -rf`/`Remove-Item -Recurse` en boucle avec "répertoire non vide". | Copie de `node_modules` vers un `git worktree` temporaire pour comparer deux états de code, sur Windows (Git Bash + PowerShell). | Après un `TaskStop`, vérifier via `ps aux` (ou `Get-Process` côté Windows) que le process a réellement disparu avant de considérer un nettoyage sûr. Pour supprimer un `node_modules` pnpm profondément imbriqué sur Windows, utiliser `robocopy "$dossierVide" "chemin\node_modules" /MIR` (qui gère les chemins longs nativement) pour le vider avant `Remove-Item`. |
