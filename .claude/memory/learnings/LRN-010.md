# LRN-010 — `pnpm lint` JSON parse EOF via rtk → vérifier un ESLint global qui masque le local

| Pattern observé                                                                                                                                                                                                                                                                                                                                                   | Contexte                                                                                                                                                                                                                                                                                                     | Application future                                                                                                                                                                                                                                                                                                                   |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Quand un hook/wrapper (`rtk` ou équivalent) fait planter `pnpm lint`/`eslint` avec une erreur de parsing JSON incompréhensible, vérifier en premier si le binaire réellement exécuté est le bon — un ESLint global installé via npm peut shadower silencieusement la version locale du projet et planter sur des règles/plugins absents de sa propre installation | `pnpm lint` (via `rtk`) échouait avec `JSON parse failed: EOF while parsing a value`. Diagnostic : `eslint --version` global renvoyait 9.9.0 alors que `node_modules/.bin/eslint --version` du projet renvoyait 10.6.0 — le global plantait sur `no-unassigned-vars`, règle absente de son propre plugin set | Comparer systématiquement `which eslint` / `eslint --version` (résolution PATH) vs `node_modules/.bin/eslint --version` (version projet) avant de chercher un bug dans le code lui-même ; contourner en invoquant directement `node_modules/.bin/eslint`, `tsc -b`, `vite build` en attendant un correctif de configuration PATH/rtk |
