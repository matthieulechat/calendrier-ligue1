# LRN-023 — `flex-1` sans parent `display:flex` = no-op silencieux

| Pattern observé                                                                                                                                                                                                     | Contexte                                                                                                                                                                                                                                                                                         | Application future                                                                                                                                                                                                                     |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Un enfant avec `flex-1`/`flex-grow` n'a strictement aucun effet si le parent n'est pas explicitement un conteneur flex (`display:flex`) — aucune erreur, aucun warning, la classe est juste silencieusement inerte. | La grille de matchs dans `PosterSheet` avait `flex-1` depuis la Phase 1 pour pousser `PosterFooter` en bas de page, mais le conteneur parent n'avait jamais `flex flex-col` — bug resté invisible tant que le contenu dépassait naturellement la hauteur de page (masquant l'absence d'ancrage). | Avant de chercher pourquoi un `flex-1`/`flex-grow` ne produit pas l'effet attendu, vérifier en premier que le parent immédiat a bien `display:flex` (ou `inline-flex`) — c'est la cause la plus fréquente et la plus rapide à écarter. |
