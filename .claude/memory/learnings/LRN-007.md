# LRN-007 — Spike multi-candidats : agents parallèles + verdict GO/NO-GO structuré identique

| Pattern observé                                                                                                                                                                                                                                                                                                                                               | Contexte                                                                                                                       | Application future                                                                                                                                                                                                                                             |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Évaluer 3 APIs candidates (openligadb, football-data.org, API-Football) sur les mêmes 5 critères (couverture, fixtures+résultats, délai, quota, fiabilité) via 3 agents lancés en parallèle avec un prompt structurellement identique (mêmes critères, même format de verdict GO/NO-GO) a permis d'obtenir une synthèse comparable, immédiatement exploitable | Phase 2 (spike de faisabilité données) du projet calendrier-ligue1 — décision go/no-go à trancher entre 3 APIs foot candidates | Pour toute évaluation de N services/APIs candidats sur les mêmes critères de décision, forker N agents en parallèle avec un prompt-template partagé (mêmes critères, même format de sortie) — garantit une comparaison apples-to-apples et une synthèse rapide |
