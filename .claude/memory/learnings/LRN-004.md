# LRN-004 — Tailwind v4 : opacité arbitraire hors du pas de 5 nécessite `/[N%]`

| Pattern observé                                                                                                                                                                                                                                                              | Contexte                                                                                                                                                                              | Application future                                                                                                                                                           |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Le modificateur d'opacité Tailwind (`bg-white/N`, `text-white/N`) n'accepte nativement que l'échelle par défaut (0, 5, 10, ..., 100) — une valeur comme `/88` ou `/18` ne résout à rien (classe ignorée silencieusement, pas d'erreur build) car ce ne sont pas des pas de 5 | Portage d'un CSS V1 précis (rgba avec des alphas comme .18, .88, .92, .93, .45) vers des classes Tailwind — plusieurs valeurs alpha réelles ne tombaient pas sur l'échelle par défaut | Pour toute valeur alpha hors de l'échelle standard, utiliser la syntaxe crochet `bg-white/[88%]` (ou `/[0.88]`) plutôt que la forme courte `/88`, qui échoue silencieusement |
