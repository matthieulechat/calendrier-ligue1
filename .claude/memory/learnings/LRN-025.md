# LRN-025 — Repo tiers luukhopman non mis à jour après un rebrand officiel ; Wikimedia Commons daté fait référence

| Pattern observé                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | Contexte                                                                                                                                                                                                                                                                                                                                                                                | Application future                                                                                                                                                                                                                                                                          |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Le repo `luukhopman/football-logos` (source des PNG dans `public/logos/ligue1/`, cf. [BDR-003](../decisions/BDR-003.md)) sert un snapshot figé au moment du clonage — il ne se met pas à jour tout seul quand un club change de blason officiellement. `marseille.png` (139×181, téléchargé le 2026-07-02) était encore l'ancien crest "Bouchet" alors que l'OM a dévoilé un nouveau logo le 2026-04-08 (effectif à partir de la saison 2026-27, celle ciblée par ce projet). Re-télécharger depuis le même repo a donné un fichier `cmp`-identique à l'existant : la source elle-même est datée, pas un problème de récupération côté projet. | Signalement Baptiste : "le dernier logo de l'OM n'est pas récupéré". Vérification : fichier local = crest bleu classique avec "M" blanc, alors que le crest 2026 (bleu "Marseille Blue" + or, étoile, "DROIT AU BUT") est déjà officiel et documenté (SportsLogos.net, Eurosport).                                                                                                      | Pour tout logo dont le club a annoncé un changement récent, ne pas se fier uniquement au repo luukhopman — comparer la date de l'annonce officielle à la date de récupération du fichier local, et re-vérifier même si le repo "semble" à jour (aucune erreur, juste un fichier identique). |
| Wikimedia Commons héberge directement l'asset SVG pour un rebrand très récent et médiatisé, avec un nom de fichier daté et versionné (`Logo_Olympique_de_Marseille_avec_étoile_(2026).svg`) — contrairement au pattern habituel [LRN-003](LRN-003.md) où le logo vit sur le wiki de langue faute d'upload sur Commons. Plusieurs variantes coexistent sur Commons pour un même club (avec/sans étoile, avec/sans motto) : chercher via `Special:Search` plutôt que deviner un seul nom de fichier.                                                                                                                                             | Recherche du nouveau blason OM : `fr.wikipedia.org` API a renvoyé "missing" pour le nom de fichier deviné directement ; passage par une recherche Commons (`Special:Search?search=Olympique+de+Marseille+logo`) a listé toutes les variantes 2026 disponibles, dont celle avec étoile+motto qui correspond à l'usage du projet (crest complet, pas la version institutionnelle épurée). | Pour un rebrand très récent, chercher d'abord sur Commons via `Special:Search` (pas seulement l'infobox de l'article) et comparer les variantes disponibles (avec/sans étoile, couleurs) avant de choisir laquelle correspond à l'usage (sportif vs institutionnel).                        |
| `sharp` (déjà dépendance du projet, cf. [BDR-006](../decisions/BDR-006.md)) rend un SVG en PNG transparent haute résolution directement (`sharp(svgPath, {density}).resize().png()`), sans passer par `agent-browser`/canvas navigateur comme dans [LRN-008](LRN-008.md).                                                                                                                                                                                                                                                                                                                                                                      | Conversion du SVG OM 2026 (184×256 viewBox) en PNG 400×556 transparent, `public/logos/ligue1/marseille.png`.                                                                                                                                                                                                                                                                            | Préférer `sharp` à la conversion navigateur pour toute conversion SVG→PNG ponctuelle dans ce projet — dépendance déjà installée, pas besoin d'ouvrir un navigateur pour ça. Réserver le pattern `agent-browser`/canvas de LRN-008 aux cas où `sharp` n'est pas disponible.                  |
