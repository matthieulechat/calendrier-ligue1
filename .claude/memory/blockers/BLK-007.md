# BLK-007 — `pnpm lint` (rtk) échoue : ESLint global masque le binaire local du projet

| Friction                                                                                                                                                    | Cause réelle                                                                                                                                                                                                                                                                   | Solution                                                                                                                                                                                                                                                                                                                                                     | Statut |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| `pnpm lint` renvoie `ESLint output (JSON parse failed: EOF while parsing a value at line 1 column 0)` via le wrapper `rtk`, sans exécuter le lint du projet | Le hook `rtk` résout un ESLint global (`~/AppData/Roaming/npm/eslint`, v9.9.0) au lieu du `node_modules/.bin/eslint` local du projet (v10.6.0) — la version globale plante sur la règle `no-unassigned-vars`, absente de son propre plugin set, et produit une sortie non-JSON | Contournement appliqué cette session : invoquer `node_modules/.bin/eslint .`, `tsc -b` et `vite build` directement (0 erreur confirmé, 1 warning pré-existant dans `button.tsx` sans lien). La cause racine (PATH/config `rtk` qui priorise le global) n'est pas corrigée — `pnpm lint` replantera tant que ce n'est pas réglé côté configuration `rtk`/PATH | ouvert |
