# BLK-002 — shadcn init : "Could not load the workspace config"

| Friction                                                                                                                                                                                                                | Cause réelle                                                                                                                                                                                                                                                            | Solution                                                                                                                             | Statut |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| `pnpm dlx shadcn@latest init` échouait systématiquement (2 versions testées, `@latest` et `4.11.0`) avec "Could not load the workspace config", alors que `components.json` venait d'être écrit avec succès juste avant | Le `tsconfig.json` racine ne contenait que `"files": []` + `references` vers `tsconfig.app.json`/`tsconfig.node.json` (projet composite), sans `compilerOptions.paths` en direct — le résolveur du CLI shadcn ne suit pas les `references` pour retrouver l'alias `@/*` | Dupliquer `compilerOptions.paths: { "@/*": ["./src/*"] }` directement dans le `tsconfig.json` racine, en plus de `tsconfig.app.json` | résolu |
