# BDR-001 — Pas de `CompetitionAdapter` avant 2e compétition réelle

| Décision                                                                                                                                                                                                                                 | Pourquoi                                                                                                                                                           | Alternatives considérées                                                                                                                                                         | Statut |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| L'adapter `src/adapters/ligue1.ts` reste concret et spécifique tant qu'une seule compétition est implémentée. Aucune interface `CompetitionAdapter` généraliste n'est conçue avant l'ajout réel d'une 2e compétition (Ligue 2, Top 14…). | Concevoir une abstraction à partir d'un seul cas concret (Ligue 1) mène à une interface mal formée, devinée plutôt qu'observée depuis de vrais besoins divergents. | Généraliser dès maintenant en anticipant les besoins de Ligue 2/Basket Pro A/Top 14 — écarté explicitement dans le Backlog de `docs/ROADMAP.md` comme sur-ingénierie prématurée. | actif  |
