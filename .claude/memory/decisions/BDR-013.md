# BDR-013 — Deploy key GitHub (repo unique) plutôt que PAT fine-grained pour le push cron Pi

| Décision                                                                                                                                                                                                                                   | Pourquoi                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | Alternatives considérées                                                                                                                                                                                                | Statut |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| Le cron PM2 (`sync-matches.ts --push`) sur le Raspberry Pi pousse vers GitHub via une **deploy key SSH dédiée** (`ed25519`, générée sur le Pi, accès write limité à `matthieulechat/calendrier-ligue1`) plutôt qu'un Personal Access Token | Une deploy key est scopée à un seul repo — même si le Pi (device partagé, plusieurs autres crons dessus) est compromis, aucun autre repo de l'organisation n'est exposé. Un PAT classique couvre tous les repos de l'utilisateur sauf configuration scope très précise. Le heartbeat Uptime Kuma déjà en place compense l'absence d'expiration automatique de la deploy key : une clé révoquée ou cassée se voit immédiatement (statut "down"), pas besoin de rotation planifiée pour détecter un problème | PAT fine-grained scopé au repo avec expiration forcée (ex. 90 jours) — équivalent en scope, mais ajoute une charge de renouvellement manuel récurrente pour un gain de sécurité marginal une fois le heartbeat en place | actif  |
