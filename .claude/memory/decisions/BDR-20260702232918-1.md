# BDR-20260702232918-1 — Phase 5 : localStorage scores par compétition/saison (pas par club) + prop-drilling

| Décision                                                                                                                                                                                                                                                                                                                                                                   | Pourquoi                                                                                                                                                                                                                                                                                                                                                   | Alternatives considérées                                                                                                                                                                                                                                                                                   | Statut |
| -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| La persistance des scores saisis utilise une clé unique `scores:{competitionId}:{season}` (pas scopée par club) indexée par `matchKey = "{journee}-{domicile}-{exterieur}"`, et la donnée fusionnée + le callback de mise à jour remontent via prop-drilling sur 4 niveaux (`App` → `PosterSheet` → `MonthBlock` → `MatchRow` → `ScoreBox`), sans Context React ni Zustand | Un match a un seul score peu importe l'affiche club depuis laquelle on le consulte — scoper par club dupliquerait/désynchroniserait la donnée. Le prop-drilling reste cohérent avec le pattern déjà en place pour `club`/`clubsById` sur la même profondeur d'arbre, pas de justification à introduire un mécanisme différent pour 2 props supplémentaires | Clé `localStorage` scopée par club (écartée : incohérente, un même match afficherait des scores différents selon le club sélectionné) ; React Context ou store Zustand pour éviter le prop-drilling (écarté tant qu'une 3e prop de config n'a pas besoin de traverser ces niveaux sans être lue en chemin) | actif  |
