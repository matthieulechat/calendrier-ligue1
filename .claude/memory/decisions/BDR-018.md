# BDR-018 — Bouton d'installation PWA : réutilisation du hook `useInstallPrompt` d'ifecho, FAB dédié

| Décision                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             | Pourquoi                                                                                                                                                                                                                                                                            | Alternatives considérées                                                                                                                                                                                                                            | Statut |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------ |
| Portage quasi à l'identique du hook `useInstallPrompt` (Android via `beforeinstallprompt` + `deferredPrompt.prompt()`, iOS via détection userAgent + hint manuel Share, guard `display-mode: standalone`) et du composant associé depuis `ifecho` (`src/hooks/useInstallPrompt.ts`, `src/components/InstallButton`). Rendu ici comme `InstallFab.tsx` (nouveau composant, pas `InstallButton`) pour matcher le pattern FAB existant du projet (`HelpFab`/`PrintFab`, `size="fab"` du bouton shadcn) plutôt que le bouton pilule du header ifecho. Positionné `fixed bottom-[136px] right-6` pour s'empiler au-dessus de `HelpFab` (`bottom-20`) sans collision. Seule adaptation : `isStandalone` dérivé via l'initializer de `useState` plutôt qu'un `useEffect` + `setState` (règle react-doctor `set-state-in-effect`, cf. [GLRN-059](../../../../../../../.claude/global-memory/learnings/GLRN-059.md)). Ajout en parallèle d'un `AppLogo.tsx` (icône `pwa-192x192.png` fixe en haut à gauche, symétrique de `AppBrandRibbon` en haut à droite). | ifecho a déjà validé ce hook en prod (cf. [GLRN-136](../../../../../../../.claude/global-memory/learnings/GLRN-136.md)) — pas de raison de le redériver. Le style FAB (pas bouton pilule) reste cohérent avec l'UI existante du projet plutôt qu'importer le style ifecho tel quel. | Bouton pilule dans le header façon ifecho (écarté — casse la cohérence visuelle avec les FAB `HelpFab`/`PrintFab` déjà en place) ; toast/bannière de notification post-install façon `vite-plugin-pwa` (jamais envisagé, hors scope de la demande). | actif  |
